日常启动只需第 2 节的一条命令。

- `swufe-rag/.venv`:**Python 3.12** 虚拟环境(uv 创建)。

  f-string 表达式含反斜杠的语法,需要 3.12+。

  (`~/.cache/huggingface/hub/models--BAAI--bge-large-zh-v1.5`)。
  本机网络下 huggingface_hub 的 Python TLS 全被掐断,是用 curl 从
  ModelScope(`AI-ModelScope/bge-large-zh-v1.5`,直连不走代理)逐文件
  下载后按 snapshot 布局放置的;离线加载已验证(dim=1024)。
- `python scripts/verify_migration_bundle.py` 静态验收通过
  (57 来源 / 35,828 课程行 / 60,827 向量 / 2024 网安 8 学分规则)。

## 2. 启动后端

```bash
cd "/Users/xuwenyao/swufe-rag/back-end engineer/swufe-rag"
.venv/bin/python -m app.server
# 就绪后: http://127.0.0.1:8000  (网页聊天 / /docs / /openapi.json)
```

首个 /ask 会加载 BGE 模型与 FAISS 索引,冷启动约十几秒;之后 SQL 路径
问题约 0.2s 返回。

## 3. iOS 客户端(client-ios/SwufeAsk)

已按 `FRONTEND_API_V16.md` 完成对接:

- `/options`:解析 `majors_by_cohort`,「提问范围」新增专业选择,按年级过滤;
  切换年级自动清掉不属于该年级的专业。
- `/ask`:请求体带 `major`;钥匙串里存有 LLM Key 时自动加 `X-LLM-API-Key`
  请求头(BYOK,在 App「关于与数据说明 › LLM Key」配置,存本机钥匙串,
  仅随请求发送)。响应新增解析 `execution_path`(课程库/文档检索标签)与
  `validation.passed`(不通过按证据不足展示)。
- `/source/{chunk_id}`:模型宽松解码,元数据标签按有无条件显示。
- 回答渲染:新增 AnswerMarkdownView,支持 V16 答案里的 Markdown 标题与
  表格(等宽横向滚动),链接(原文件页/下载)可点击。

模拟器直连 127.0.0.1 即可;真机在「关于 › 后端地址」填 Mac 局域网地址。

## 4. 已验证的联调路径(2026-07-18,模拟器)

- 范围:2024 级 + 网络空间安全专业(专业列表来自后端 56 项)

  毕业最低 **152 学分**、专业选修模块最低 **8 学分**,引用 2024 总册
