# mock/ —— 模块D桩数据与桩实现

**⚠️ 本目录全部内容为演示用假数据(桩),不是西南财经大学的真实文件内容。**

模块A(数据)、模块B(检索)、模块C(生成)交付前,模块D用本目录数据独立开发前端与评估框架;集成日把 `config.yaml` 的 `provider: mock` 改为 `real` 即弃用本目录(目录保留,便于离线演示兜底与回归测试)。

## 数据说明

| 文件 | 内容 | 契约 |
|------|------|------|
| `mock_chunks.jsonl` | 24 条手造知识块,字段严格按契约1 | 契约1 |
| `mock_answers.json` | 17 组手造问答对 + 拒答模板,格式严格按契约3 | 契约3 |
| `mock_provider.py` | `MockRetriever`(真实过滤逻辑+简易文本相似度)与 `MockGenerator`(关键词匹配预制答案) | 契约2/3 签名 |

## 编造原则(诚实声明)

- 条款内容**参考真实教务文件的行文风格编造**,数字(如"165学分""前30%""425分")均为虚构演示值,不得作为真实政策依据
- `page_url` / `file_url` 指向**真实存在的官网栏目入口页**(如 jwc.swufe.edu.cn、it.swufe.edu.cn),便于演示"查看官网原文"跳转;真实数据由模块A登记实际通知页与附件直链后替换
- 刻意覆盖前端所有展示分支:校级+院级、两个学院(计算机与人工智能学院/金融学院)、两个年级(2022/2023)、现行+历史版本、`is_table=true` 表格块 ×3、超长条款(school_xjgd_007)、单引用/多引用/跨文件引用、拒答、适用范围提醒、同题不同院差异化答案

## mock_answers.json 的 matching 字段

`keywords`(主题词)/`keywords2`(限定词)/`min_hits`/`college`/`cohort` 五个字段仅供 `MockGenerator` 做关键词路由,**不属于契约3**,真实模块C不需要它们。匹配规则见 `mock_provider.py` 注释。
